Closures

Was sind Closures?

Closures sind Funktionen mit unabhängigen,
freien Variablen.

Die in der Closure definierte Funktion merkt sich die Umgebung, in der sie erzeugt wurde.

In einfachen Worten ist eine Closure nichts anderes als eine Funktion, die in einer anderen Funktion definiert ist. Die Closure hat Zugriff auf alle lokalen Variablen, externen Funktionsvariablen und globalen Variablen. Sie ist eine Funktion, die mit ihrer lexikalischen Umgebung gebündelt ist.

Closures in simple words:

Closures sind Funktionen mit konservierten Werten.

Wie wir Closures einsetzen können

Closures anstelle von Objekten: Funktionen mit Kontext erstellen
Wir können ganz einfach eine Closure erstellen, bei der alle allgemeinen Felder von der äußeren Funktion festgelegt werden und die innere Funktion nur die eindeutigen Felder verwendet: Dies kann in einigen Fällen sehr nützlich sein z.B. wenn wir mehrere Funktionen benötigen, die in unterschiedlichen Kontexten etwas Ähnliches tun. Anstatt für jede Aktion eine eigene Funktion zu erstellen, erstellen wir eine äußere Funktion und mehrere innere Funktionen. Diese innere Funktionen "returnen" wir dann an die übergeordnete Funktion zurück.
Hinweis: Return-Funktionen sind nichts anderes als Closures.

Private Attribute erstellen
Sprachen wie Java geben uns die Möglichkeit, bestimmte Attribute (Eigenschaften) als "private" zu deklarieren. Dies ist bei JavaScript (oder Python) nicht der Fall. Aber das wir können dies durch Folgendes erzwingen: z.B. Die Person-Funktion gibt ein Objekt mit Funktionen zurück, die von außerhalb der Person-Funktion zugänglich sind. Dies ist über die Punktnotation (d. h. john.grow())möglich. Neben diesen Funktionen hat das Objekten aber zwei Attribute, die sie nicht über "return" ausgibt - deshalb bleiben diese außerhalb der Funktion Person unzugänglich.

    Hier ein paar weitere Use-Cases:
  • Module design pattern
  • Event handlers
  • Asynchronous Programming
  • Currying (Currying in Javascript ist nichts anderes als das Ändern einer Funktion mit mehreren Argumenten in eine Funktion, die nur ein Argument benötigt.)
  • Memoization (Ist die kurzfristige Speicherung einer bereits erledigten Abfrage und Antwort. Diese Technik wird verwendet, um aufwändige Berechnungen zu vermeiden.)

Closures haben Zugriff auf:

  • Ihren lokaler Geltungsbereich (Variablen, die in der inneren Funktion definiert sind)
  • den Umfang ihrer äußeren Funktionen
  • Den globale Geltungsbereich
