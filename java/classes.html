Datentypen

In Programmiersprachen gibt es verschiedene Arten von Daten mit denen gearbeitet wird.

identifier: Name der Variablen, Klassen, Methoden, Funktion - dies sind alle sogenannte "Bezeichner". Wenn wir also eine Variable deklarieren, ist ein Variablenname ein Bezeichner/identifier.

literal: Dies sind die syntaktischen Darstellungen unserer Daten. Einfach gesagt, sind dies die tatsächlichen Werte unserer Variablen.

Primitive Datentypen (Primitives)
Primitive Datentypen in Java werden von der Java-Sprache vordefiniert und als reservierte Schlüsselwörter benannt. Ein primitiver Datentyp teilt keinen Zustand (State) mit anderen primitiven Werten.


Schauen wir uns alle 8 primitiven Datentypen an, die von Java unterstützt werden.

  • byte: (byte theByte=10;)
  • short: (short theShort=10;)
  • int: zum Speichern von Dezimalzahlen(int theInteger=10;)
  • long: um große ganzzahlige Werte zu speichern (long theLong=5_000_000l;)
  • float: zum Speichern von Dezimalzahlen (float theFloat=10f;)
  • double: um große Dezimalzahlen zu speichern (double theDouble=10.1010;)
  • boolean: entweder true oder false (boolean theBoolean=1;) nützlich für bedingte Anweisungen
  • char: um ein einzelnes Zeichen oder einen Buchstaben zu speichern (char theChar ='s';)
Nicht-primitive Datentypen (References)
Nicht primitive Datentypen werden Referenztypen genannt, weil sie Objekte referenzieren bzw. auf Objekte verweisen. Im Gegensatz zu primitiven Datentypen, die "by value" kopiert werden, verweist eine Kopie eines nicht-primitiven Datentyps immer auf dieselbe Addresse und deren Inhalt. Bei der Übergabe der Referenzvariable wird die Referenz (der Verweis auf das Objekt) kopiert und in der Parametervariable der Methode gespeichert. Innerhalb der Methode wird nun mit einer Kopie der Referenz gearbeitet, dadurch erfolgt der Zugriff direkt auf das Originalobjekt.

Es gibt viele Referencetypes in Java: Strings, Arrays, Klassen usw.
Sehen wir uns an, wie man ein Array und einen String initialisiert.
  • Array: Eine lineare Datenstruktur die Werte an zusammenhängenden Speicherorten speichert. (int[] theArray = new int[10];)
  • String: Der Datentyp zum Arbeiten mit Text (String theString = "hello world";)
Typkonvertierung & Casting
Wenn wir versuchen, eine Dezimalzahl oder einen Gleitkommawert in eine String-Variable einzugeben, erhalten wir eine Fehlermeldung. Dasselbe gilt, wenn wir eine Gleitkommazahl in eine Variable vom Typ "int" eingeben. Wenn wir andererseits eine Dezimalzahl für Float-Daten angeben, wird die Dezimalzahl automatisch in Float konvertiert.

Bedingungen für die Typkonvertierung:
Zwei Datentypen sollten für die Konvertierung kompatibel sein (z.B. char & String oder int & float). Wichtig ist zudem, dass der Zieltyp größer als der Quelltyp ist (zum Beispiel ist float größer als int).

Variablen und Objekte

Variablen
Eine Variable ist ein Name für eine Speicherstelle im Hauptspeicher, die einen bestimmten Datentyp verwendet, um einen Wert zu erhalten.
Man könnte sich eine Variable mit einem primitiven Datentyp als eine kleine, mit einem Namen versehene Box vorstellen. In dieser Box wird ein bestimmter Gegenstand einer vordefinierten Art eingelagert. Durch das Aufrufen des Namens der Box, können wir sehen, was sich in ihr befindet.

Bei Variablen geht es vor allem ums Deklarieren (Benennen) und Initialisieren (Wert zuweisen). Wichtig ist das die Variable immer erst deklariert wird, da der Compiler sonst den Variablennamen nicht erkennt und es zu einem Fehler kommt.

Variablendeklaration( Datentyp, Variablenname, Semikolon):
int myVariable;

Es ist auch möglich der Java Variablen schon während der Deklaration einen Wert zu zuweisen, also diese zu initialisieren:
datentyp variable = wert;
int myVariable = 5;

Das Besondere an Variablen in Java, ist dass wir ihnen immer wieder einen neuen Wert zuweisen können.

Objekte
Was ist ein Objekt?
Die Grundeinheit von OOP/ Eine Instanz einer Klasse.

Konstruktor Objekte können über den Konstruktor aus der Klasse erstellt werden. Es gibt 2 Arten Default / nicht parametriert (0 Argumente) Parametriert Konstruktoren müssen eine bestimmte Anzahl von Argumenten übergeben Ein Konstruktor kann nicht abstrakt final oder statisch sein

Klassen und Methoden

Klassen
Was ist eine Klasse?
Eine Vorlage(blueprint)zum Erstellen mehrerer Objekte.Sie definiert den Zustand des Objekts.


Was ist ein Zustand/State?
- Verhalten, das ein Objekt aufweisen kann.
- Zustandswerte die ein Objekt hat. In Java wären dies die Instanzvariablen des Objekts
- Verhaltensmethoden, die Teil der Klasse sind. Der Zustand des Objekts kann sich mit der Zeit/ über Verhaltens- /Klassenmethoden ändern.
Methoden
In Java spielen Methoden eine zentrale Rolle. Sie bestimmen das Verhalten von Objekten und somit auch das Verhalten des gesamten Programms. Methoden arbeiten in Java immer mit den Daten (Variablen) von Objekten. Das unterscheidet sie von Funktionen, die man in anderen Programmiersprachen vorfinden kann und welche vollkommen unabhängig von einem Objekt oder einer Klasse existieren. Die Methodendefinition wird innerhalb der Klassendefinition vorgenommen. Die Methoden haben dadurch Zugriff auf alle Variablen des Objekts.

    Methoden bestehen aus:
  • Modifier (z.B. private/ public): Häufig wird einer der Modifier private, public, default oder protected in Java benutzt.
  • Rückgabe-Datentyp
  • Methodenname
  • Parameter(-liste)
  • Methodenkörper/ Logik
    • public String myMethod(int count, string fruit){ }
Methodensignatur
Die Methodensignatur enthält einen Methodennamen(und die Argument-Datentypen) Wir können Methoden überladen, indem wir den gleichen Methodennamen benutzen aber unterschiedliche Parameter. Zwei Methoden welche den gleichen Namen haben, müssen sich nämlich in der Signatur unterscheiden. Das heißt die Parameter dürfen nicht dieselben sein. Sonst kommt es zu einem Fehler. Methodenaufruf
1. Methode steht in derselben Klasse, 2. Methode eines Objekts aufrufen mit dem Punktoperator. Vor diesem steht das Objekt, auf das die Methode angewandt werden soll und nach ihm die Methode mit entsprechendem Parameter. Wichtig ist die Signatur der Methode. Diese darf pro Klasse nur einmalig sein, denn sonst findet das Programm die Methode nicht.

Schlüsselbegriffe
Hier wird auf einige wichtige Schlüsselwörter der Java Syntax eingegangen

private/public/protected
private method: Auf die private Methode kann außerhalb der aktuellen Klasse nicht zugegriffen werden. Untergeordnete Klassen können nicht auf private abstrakte Methoden zugreifen und sie nicht implementieren. Auf public Methoden kann von überall aus zugegriffen werden.
private class: Nur innerhalb der Klasse kann auf eine verschachtelte private Klasse zugegriffen werden.
Bei protected können wir im selben Java Package oder einer Sub-Klasse darauf zugreifen.

static
static variable: Eine einzelne Kopie der Variablen wird erstellt und von allen Objekten auf Klassenebene gemeinsam genutzt
static method: Die Deklaration "static" zeigt an dass die Methode zu einer Klasse gehört. Sie wird verwendet, um Klassenmitglieder unabhängig von Instanzen zu verwenden (es muss also kein Objekt instanziiert werden, um die Methode zu verwenden).
Weitere Eigenschaften:
- Kann vor allen erstellten Objekten der Klasse und ohne Bezug auf ein Objekt zugegriffen werden.
- kann nur andere statische Methoden direkt aufrufen
- kann nur direkt auf statische Daten zugreifen
- kann ausgeblendet, aber nicht überschrieben werden
- können nicht abstrakt sein, da abstrakte Methoden überschrieben werden müssen, um implementiert zu werden.

final
final variable: Um eine konstante Variable zu erstellen.
Wert kann nicht verändert werden.
Kann im Konstruktor initialisiert werden.
final method:
Um das Überschreiben der Methode zu verhindern.
Kann vererbt, aber nicht überschrieben werden.
Ihr Konstruktor kann nicht vererbt, also auch nicht überschrieben werden und hat einen Kompilierzeitfehler.
final class:
um Vererbung zu verhindern

abstract
abstract method:
Eine abstrakte Methode enthält keinen Body. Unterklassen implementieren sie. Jede Klasse, die abstrakte Methoden enthält, muss als abstract deklariert werden. Die folgenden Kombinationen von Modifikatoren können nicht mit abstract verwendet werden:
- final (Klasse & Methode)
- private (Methode)
- static (Methode)
- synchronisiert (Methode)
- native / strictfp

abstract class:
Objekt der abstrakten Klasse kann nicht erstellt/instanziiert werden. Jede Unterklasse der abstrakten Klasse muss entweder alle abstrakten Methoden implementieren oder selbst als abstrakt deklariert werden. Aufgrund der teilweisen Implementierung von Klassen kann das Objekt nicht instanziiert werden. Die abstrakte Klasse hängt von der Unterklasse ab, um eine vollständige Implementierung bereitzustellen.

Rekursion...auch spannend!


Unter Rekursion versteht man in der Programmierung ein Verfahren, bei dem sich eine Methode selbst aufruft, sodass, ähnlich einer Endlosschleife, ein potentiell unendlicher Programmablauf entsteht.
