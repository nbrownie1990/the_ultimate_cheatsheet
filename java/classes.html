Datentypen

In Programmiersprachen gibt es verschiedene Arten von Daten mit denen gearbeitet wird.

Primitive Datentypen (Primitives)
Primitive Datentypen in Java werden von der Java-Sprache vordefiniert und als reservierte Schlüsselwörter benannt. Ein primitiver Datentyp teilt keinen Zustand (State) mit anderen primitiven Werten.
Man könnte sich eine Variable mit einem primitiven Datentyp als eine kleine, mit einem Namen versehene Box vorstellen. In dieser Box wird ein bestimmter Gegenstand einer vordefinierten Art eingelagert. Durch das Aufrufen des Namens der Box, können wir sehen, was sich in ihr befindet.

Schauen wir uns alle 8 primitiven Datentypen an, die von Java unterstützt werden.

  • byte: (byte theByte=10;)
  • short: (short theShort=10;)
  • int: zum Speichern von Dezimalzahlen(int theInteger=10;)
  • long: um große ganzzahlige Werte zu speichern (long theLong=5_000_000l;)
  • float: zum Speichern von Dezimalzahlen (float theFloat=10f;)
  • double: um große Dezimalzahlen zu speichern (double theDouble=10.1010;)
  • boolean: entweder true oder false (boolean theBoolean=1;) nützlich für bedingte Anweisungen
  • char: um ein einzelnes Zeichen oder einen Buchstaben zu speichern (char theChar ='s';)
Nicht-primitive Datentypen (References)
Nicht primitive Datentypen werden Referenztypen genannt, weil sie Objekte referenzieren bzw. auf Objekte verweisen. Im Gegensatz zu primitiven Datentypen, die "by value" kopiert werden, verweist eine Kopie eines nicht-primitiven Datentyps immer auf dieselbe Addresse und deren Inhalt.

Es gibt viele Referencetypes in Java: Strings, Arrays, Klassen usw.
Sehen wir uns an, wie man ein Array und einen String initialisiert.
  • Array: Eine lineare Datenstruktur die Werte an zusammenhängenden Speicherorten speichert. (int[] theArray = new int[10];)
  • String: Der Datentyp zum Arbeiten mit Text (String theString = "hello world";)
identifier: Name der Variablen, Klassen, Methoden, Funktion - dies sind alle sogenannte "Bezeichner". Wenn wir also eine Variable deklarieren, ist ein Variablenname ein Bezeichner/identifier.

literal: Dies sind die syntaktischen Darstellungen unserer Daten. Einfach gesagt, sind dies die tatsächlichen Werte unserer Variablen.

Typkonvertierung & Casting
Wenn wir versuchen, eine Dezimalzahl oder einen Gleitkommawert in eine String-Variable einzugeben, erhalten wir eine Fehlermeldung und das Programm schlägt fehl. Dasselbe gilt, wenn wir eine Gleitkommazahl in eine Variable vom Typ "int" eingeben. Wenn wir andererseits eine Dezimalzahl für Float-Daten angeben, wird die Dezimalzahl automatisch in Float konvertiert.

Bedingungen für die Typkonvertierung:
Zwei Datentypen sollten für die Konvertierung kompatibel sein (z.B. char & String oder int & float). Wichtig ist auch, dass der Zieltyp größer als der Quelltyp ist (zum Beispiel ist float größer als int).
